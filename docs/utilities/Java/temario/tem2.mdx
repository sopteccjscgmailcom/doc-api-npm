# 🚀 Patrones de Desarrollo de Software

Los **patrones de desarrollo de software** son soluciones **reutilizables** y **probadas** para problemas comunes en la arquitectura y diseño de software. Estos patrones ayudan a escribir código más estructurado, modular y mantenible.  

💡 **Piensa en ellos como "recetas" de diseño que pueden aplicarse a diferentes situaciones en la programación.**  

---  

### **📌 Clasificación de los Patrones de Diseño**  

🔹 **1. Patrones Creacionales** 🏗️ *(Cómo crear objetos de manera eficiente y flexible)*  
🔹 **2. Patrones Estructurales** 🏛️ *(Cómo organizar clases y objetos para formar estructuras más grandes)*  
🔹 **3. Patrones de Comportamiento** 🔄 *(Cómo gestionar la comunicación entre objetos y su interacción)*  

---

## **🛠️ 1. Patrones Creacionales** *(Controlan la creación de objetos)*
💡 **Ejemplo:** Evitar crear objetos innecesarios o centralizar la creación en una sola clase.  

✅ **Ejemplos de patrones creacionales en Java:**  
- **Singleton** 🔒 → Garantiza que haya **solo una instancia** de una clase.  
- **Factory Method** 🏭 → Permite crear objetos sin especificar su tipo exacto.  
- **Abstract Factory** 🏗️ → Crea **familias de objetos relacionados** sin especificar sus clases concretas.  
- **Builder** 🏠 → Facilita la creación de objetos **complejos paso a paso**.  
- **Prototype** 🎭 → Clona objetos en lugar de crearlos desde cero.  

---

## **🏛️ 2. Patrones Estructurales** *(Organizan la estructura del código)*
💡 **Ejemplo:** Cómo conectar clases sin que estén demasiado acopladas.  

✅ **Ejemplos de patrones estructurales en Java:**  
- **Adapter** 🔌 → Permite que dos clases incompatibles trabajen juntas.  
- **Bridge** 🌉 → Separa una abstracción de su implementación para mayor flexibilidad.  
- **Composite** 🌲 → Organiza objetos en **estructuras de árbol** (útil para jerarquías).  
- **Decorator** 🎨 → Agrega funcionalidad a objetos sin modificar su estructura.  
- **Facade** 🏢 → Crea una interfaz simplificada para un sistema complejo.  
- **Proxy** 🕵️‍♂️ → Controla el acceso a un objeto, como un intermediario.  

---

## **🔄 3. Patrones de Comportamiento** *(Gestionan la comunicación entre objetos)*
💡 **Ejemplo:** Cómo hacer que los objetos interactúen sin crear dependencias rígidas.  

✅ **Ejemplos de patrones de comportamiento en Java:**  
- **Observer** 👀 → Permite que un objeto notifique a otros cuando su estado cambia.  
- **Strategy** 🎭 → Permite cambiar el comportamiento de un objeto en tiempo de ejecución.  
- **Command** 📜 → Encapsula una acción en un objeto, útil para deshacer acciones.  
- **State** 🚦 → Cambia el comportamiento de un objeto según su estado interno.  
- **Chain of Responsibility** 🔗 → Pasa una solicitud a través de una cadena de manejadores.  
- **Mediator** 🏢 → Reduce dependencias entre objetos al centralizar la comunicación.  
- **Visitor** 🚶‍♂️ → Permite agregar nuevas operaciones sin modificar las clases existentes.  

---

### **🎯 Resumen**
| Tipo | Propósito | Ejemplo |
|------|----------|---------|
| **Creacionales** | Controlan la creación de objetos | `Singleton`, `Factory Method` |
| **Estructurales** | Organizan la estructura del código | `Adapter`, `Decorator` |
| **Comportamiento** | Gestionan la comunicación entre objetos | `Observer`, `Strategy` |
