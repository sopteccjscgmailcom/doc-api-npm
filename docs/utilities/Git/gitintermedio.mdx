## **ðŸš¨ 3. ResoluciÃ³n de Conflictos**  

### **ðŸ”¹ Tipos de conflictos comunes**  
1. **Merge conflicts**: Al unir ramas con cambios contradictorios.  
2. **Rebase conflicts**: Al reordenar commits.  

### **ðŸ”¹ Pasos para resolver conflictos**  
1. Identifica archivos en conflicto (`git status`).  
2. Edita manualmente (busca `<<<<<<<`, `=======`, `>>>>>>>`).  
3. Marca como resuelto:  
   ```bash
   git add archivo-resuelto.txt
   git commit  # Si es merge
   git rebase --continue  # Si es rebase
   ```

### **ðŸ”¹ Herramientas visuales**  
```bash

```
